Deployment Automation Utility

  • In a typical SmartCOMM development project, code needs to be migrated between different tenancies/staging servers. The project team need to manually export and import code by logging into different tenancies and manage physical files (package file). This task takes lot of effort to get migrate code and dependency of lead developers.
  • In order to automate this process and we have developed this utility using Python, which Export & Import Projects/Resources from one SmartCOMM Tenancy to other tenancy.




  • One click Code Migration – reduces multiple steps involved in the process.
    • ​
  • Simple to use and can be managed by associates of any skill set and its one-time configuration.
  • Accuracy – Eliminates human errors due to negligence or oversight.

  • Before the demo, we should have at least few released projects in source and destination tenancies
  • We will walk the audience through the front end to explain how to configure the utility
  • For demo purpose we will configure our na4 and preview tenancies (can be accessed from any network) with released projects
  • Projects/Resources will be listed as per selected by resource IDs
  • Once user selects the project/resource and click on migrate, following APIs will be called and project/resource is migrated to destination tenancy.

    •      1. Export
         2. Import
         3. Conflict

  • Upon completion of the process, user will be prompted to check log information.
  • User can refer the logs for conflict details/successful migration
  • As next steps we will walk them through the main code which will enable the audience to understand the code at high level.
  • The Demo time is 30 min to go over the kit and all the flow
