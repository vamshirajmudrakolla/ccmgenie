Custom Java Connector - Communication Builder​


Communication server has a number of built-in connectors that serve multiple purposes. Custom Java connector is an advanced feature in Communication Server that allows users to write their own routines e.g. a) moving files between processes b) upload outputs to servers etc..​ ​

We have developed a Custom Java connector which enables integration of Communication Server with any REST API services. ​

Based on our experience working in multiple projects, we observed that there is a necessity of an utility that can import an MPW file to the Empower console, without giving controls to the main program/ calling service. ​

This connector utility does that function by establishing connectivity with the Empower APIs and uploading the MPW file, directly from the communication server jobs. ​

  • This connector can be enhanced to interact with other REST APIs by some simple modifications to the code​​
  • The inline documentation within the program is self-explanatory and can guide the developer to use the code effectively.​
  • Reuse of this code helps in saving of development and testing time.​
  • We have developed the code based on use cases from our experiences in various engagements where the Communication Server Connector integration consumed     considerable amount of effort in development and testing.​​
  • These connectors eliminate the dependency on middle-ware programs.​​




  • This is an alternate way of connectivity to any Rest APIs that will not require ESB, by using basic Java knowledge by any standard CCM developer​
  • Effort savings :- ​
    • A standard development effort for each connector takes between 2-4 Person weeks
    • By reusing this utility, the development effort could be brought down to about 1 Person week​
  • Can be enhanced further to build additional features faster​

  • Exstream installation must either install or be pointed to an existing JRE installation Communications Server application running Java connectors must be Java enabled in Control Center The StrsService.dll, StrsInDataQueue.dll, javaframework.dll files must be installed in Exstream install\Platform\Core\version\bin and jstrs.jar in Exstream install\Platform\Core\version\bin\javaclasses\core
  • The Exstream install\Applications\StreamServer\version\Tools\System\Java folder must exist The JavaConfigurator.dll file must be installed in Exstream install\Applications\StreamServer\version\Tools\System
