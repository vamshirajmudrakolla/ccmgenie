Test Data Extraction from the Test bed​


In the typical CCM world, as an industry standard, payload is XML. There will be number of scenarios to test across the templates for which team needs different XMLs with varying test data to be present. To create such XMLs, it would take considerable amount of time. ​

In order to minimize the time required in mocking the XMLs manually, ValueMomentum has created a utility which helps in passing the required test data to the XMLs from the front end to create a Mock XML that will help in testing the scenario in the template. ​ ​

This utility has been designed in such a way that it can read any XML structure and display the elements in the front end dynamically for the user to select test criteria and save it for future use. This design helps in using the utility across any project without any configuration settings.​ ​


Asset Work flow:


  • Create any number of Mock XMLs based on the requirement without manually altering the existing XML. 


    • Softwares required - java(jdk 1.8v), tomcat 9.0.
    • Languages used - html, css, jsp, java, servlets, javascript, jquery, ajax, bootstrap.
    • Files required - A base XML to identify its structure.
    • Jars required - Saxon jar files(saxon9he, saxon9-test and saxon9-xqj).
    • Folder structure to be created to store crieria files.
    • TestBed(Group of xmls) needs to be present and the path to it needs to be configured in the code.
  • It only accepts xml file as an input.
  • Folder structure is mandatory as we are not using any database.
  • While creating search criteria, logical grouping of conditions is not available.
