Command Center to Communications Builder Migration

  • OT is going to discontinue the support of Command Centre. The job scheduling (currently being done by Command Centre) can be done by using Communication Builder module of OT. Communications Builder provides a larger framework that allows to connect with multiple processes to generate output. The jobs from Command Centre need to be migrated to Communication Builder. ​
    • ​
  • This utility is developed to handle this migration automatically. UiPath (RPA tool) has been used. UiPath offers a way of automating repeated tasks without any manual intervention.
    • ​
  • This automation results into saving of ~50% of the manual effort required for migration.

  • This automation reduces ~ 50% of migration effort from CC to CB​
  • It is a low-code platform, having a set of activities, that can be easily applied to several scenarios ​

  • Job XML exported from Command Center.​​​​​
  • UiPath with project configured.​
  • Communications Builder and Control Center​


    • The demo is for ~30 minutes and takes the audience through the steps to migrating a job from Command Center to Communications Builder. Steps as below:​

    • Generate the XML for an exisiting job in Command Center. This can be done by exporting the job in Command Center, ​
    • Execute the robot that reads the Job XML and trigger a series of sequences that will replicate functionalities present in the XML into actual processes in Communications Builder (The entire process will run in the foreground enabling us to see exact mapping of functionalities)​
    • Once the project is created, generate a release which will be read by Control Center service application.​
    • Control Center will start this project as a windows service that will act as a trigger for the process.​​
