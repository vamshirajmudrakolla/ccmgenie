Exstream Package Compare Utility​


In Agile Delivery process, frequent deployments are done - at least once, every month. Deployment validation is carried out for each deployment, and, the process gets more and more complex, sprint after sprint. The deploy-ready code is compared against the previous version to ensure that only the desired components get moved. This is a manual process, and, hence, time-consuming and prone to error. ​

We have developed this simple-to-use utility to compare the components and report the exceptions so that an error free code is promoted, faster. It also produces a graphical report for easier understanding, at a component level. This utility can run in any version of OT-Exstream seamlessly. ​ ​

The utility compares two MS-Excel files created from two package files. It generates a report highlighting the differences at component level. The user is expected to confirm the highlighted changes and take appropriate action before the deployment.




  • Automates manual comparison of package reports and avoids human errors like misinterpretation or any overlook.​​
  • Effort Saving:​​​
  • Accuracy, the report generated is precise and gives a visual representation was well as detailed analysis.​​​

  • Desktop version of MS Excel
  • Package Report files in CSV format
  • As Package Report files are taken as Input
  • Approval status of an object cannot be tracked
  • Generate the package reports using Exstream Design Manager for an application​​​
  • Generate the package reports for the new version using Exstream Design Manager​​​​
  • Execute the macro to generate the comparison report as a dashboard in Excel.​
  • “Instructions To Use” sheet in the Utility explains how to use the Utility in detail​​​​
